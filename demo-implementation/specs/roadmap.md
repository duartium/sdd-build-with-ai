# Roadmap

> Línea base versionada · Estado al 2026-06-27
> Ordenado por: base técnica primero, luego valor de negocio

[ ] Fase 0 — Scaffolding: proyecto API y SPA creados, compilando y conectados a SQL Server existente
[ ] Fase 1 — Dashboard de ventas público: endpoint y vista de ventas del día sin autenticación (modo kiosco)
[ ] Fase 2 — Autenticación JWT: login con SP_USER_AUTHENTICATION, rutas protegidas y sesión persistente en el cliente
[ ] Fase 3 — Resumen de caja: endpoint y vista de aperturas, cierres y totales del día
[ ] Fase 4 — Consulta de inventario: endpoint y vista de stock actual de productos
[ ] Fase 5 — Alertas y notificaciones: detección de faltantes, cierres pendientes y umbrales por polling

## Criterios de completitud por fase
Una fase se considera Done cuando:
1. La funcionalidad corre sin errores
2. Las reglas de negocio del alcance están cubiertas
3. El código fue revisado y aprobado por el desarrollador
